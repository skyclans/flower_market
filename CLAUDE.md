# CLAUDE.md

> 이 파일은 Claude Code(CLI)가 이 저장소에서 작업할 때 읽는 컨텍스트입니다.

## 프로젝트

**꽃장부(Flower Ledger)** — 오프라인 화원용 버티컬 SaaS. 핵심 가치가설: 화원 사장이 "양재 도매 경매가 vs 내 매입가"를 즉시 비교하는 단일 도구에 돈을 낸다. MVP는 세무/환급 연동을 **명시적으로 제외**한다.

이 저장소는 그 중 가장 치명적인 리스크(PRD R1=치명)인 **양재 경매가 데이터 파이프라인**을 검증한 Phase 0 PoC다. Exit Criteria(ETL ≥90%)를 양재 100%로 통과한 상태.

## 현재 상태

- ✅ Phase 0 데이터 파이프라인 검증 완료 (de-risked)
- ✅ 양재 1년치 적재: 228,550행 / 573품목 (Neon, 2025-06-21~2026-06-20)
- ✅ Phase 1: 소스 추상화 + DB upsert + 일배치 + FastAPI 시세 API + React 프론트 (Neon 연결)
- ✅ **장부(거래 기록) v1**: 거래/품목라인 테이블 + 매입 시점 시세 스냅샷(마진 자동계산)
  - 백엔드: `/quote`(실시간 마진), `/transactions`(기록·조회), `/reports/summary`
  - 프론트: PRD IA 5탭(홈·거래·시세·리포트·더보기), 거래 입력(실시간 마진), 장부 목록, 리포트(CSV 내보내기→세무사)
- ⬜ OCR 영수증 자동 분개 (PRD P0, 외부 API 연동 필요)
- ⬜ PC 전용 레이아웃(사이드바·다열 테이블) — 현재는 반응형까지
- ⬜ 멀티유저 인증 (현재 단일 'demo' 사용자)

## 제품 정체성 (중요)
꽃장부는 **가계부(장부) + 시세 인텔리전스**다 (PRD 2.1). 시세 비교는 매일 여는 '미끼'고,
핵심 자산은 **거래 장부**(뭘 샀는지 기록 → 영수증 분실/공제 누락 문제 해결 → 추후 세무사 연계).
PRD 8.1 IA: 홈/거래/시세/리포트/더보기. North Star = WATU(주 5건 이상 기록 사용자).

## 아키텍처 결정 (지킬 것)

1. **데이터 소스**: 현재는 `flower.at.or.kr` 엑셀 다운로드(`/excel/excelDownLoad.do`). 프로덕션은 공공데이터포털 Open API `15141808`(전국 공영도매시장 실시간 경매정보)로 이관 권장 — serviceKey 필요.
2. **부류 분류**: `etl/classify.py` 의 결정론적 분류기. 키워드(난/관엽) 우선, 나머지 절화 기본값. ML 쓰지 말 것 — 결정론 유지. `classify_method` 가 신뢰 신호.
3. **권위 소스**: 일자별 경매동향 엑셀이 완전한 단일 소스. `getSaleData.json`(top-30 위젯)은 쓰지 않는다.
4. **단위/경매구분 파생**: 절화=속, 그 외=분. 월·수·금=본경매, 화·목·토=온라인. 캘린더는 `etl/codes.py`.
5. **스키마**: `schema/auction_price.sql` 가 단일 소스 오브 트루스. 컬럼 변경 시 PRD §9.3 와 동기화.
